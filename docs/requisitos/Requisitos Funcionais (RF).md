***Requisitos Funcionais (RF)***

**Os requisitos funcionais descrevem o que o sistema deve fazer.**  
**Autenticação e Usuários**

**RF01:** O sistema deve permitir o cadastro de usuários.  
**RF02:** O sistema deve permitir que usuários realizem login utilizando e-mail e senha.  
**RF03:** O sistema deve autenticar usuários por meio de token JWT.  
RF04: O sistema deve permitir a visualização e edição do perfil do usuário.  
Gerenciamento de Eventos  
**RF05:** O sistema deve permitir a criação de eventos.  
**RF06:** O sistema deve permitir a edição de eventos.  
**RF07:** O sistema deve permitir a exclusão de eventos.  
**RF08:** O sistema deve permitir adicionar uma foto de capa ao evento.  
**RF09:** O sistema deve permitir visualizar os detalhes de um evento.  
**RF10:** O sistema deve permitir pesquisar eventos por categoria.  
Convites e Participantes  
**RF11:** O sistema deve gerar links de convite compartilháveis.  
**RF12:** O sistema deve permitir convidar usuários por nome ou e-mail.  
**RF13:** O sistema deve permitir o envio de convites por e-mail.  
**RF14:** O sistema deve permitir aceitar ou recusar convites.  
**RF15:** O sistema deve exibir notificações de convites pendentes.  
**RF16:** O organizador deve poder adicionar ou remover participantes do evento.  
**RF17:** O sistema deve controlar os níveis de acesso dos participantes.  
Gestão Financeira  
**RF18:** O sistema deve registrar gastos relacionados ao evento.  
**RF19:** O sistema deve calcular automaticamente o custo por participante.  
**RF20:** O sistema deve permitir marcar pagamentos realizados pelos participantes.  
**RF21:** O sistema deve exibir débitos pendentes do evento.  
Organização de Tarefas  
**RF22:** O sistema deve permitir criar itens em um checklist.  
**RF23:** O sistema deve permitir marcar tarefas como concluídas.  
**RF24:** O sistema deve permitir atribuir tarefas aos participantes.  
Estatísticas e Informações  
**RF25:** O sistema deve exibir estatísticas relacionadas aos eventos do usuário.  
**RF26:** O sistema deve permitir atualizar as informações das telas por meio do recurso de pull-to-refresh.
