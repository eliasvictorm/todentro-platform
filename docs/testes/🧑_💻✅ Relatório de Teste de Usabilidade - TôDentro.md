# **🧑‍💻✅ Relatório de Teste de Usabilidade \- TôDentro**

## **🎯 Objetivo do teste**

| Item | Preencher ✍️ |
| :---- | :---- |
| **🎯 Objetivo** | Avaliar a facilidade de uso do aplicativo TÔDentro para criação e gerenciamento de eventos em grupo.  |
| **✅ Hipóteses** | Os usuários conseguirão criar um evento, convidar participantes, confirmar presença e visualizar informações do evento sem auxílio externo.  |
| **📌 Escopo** | Login, criação de evento, convite de participantes, checklist do rolê, visualização de participantes. |

## **🧠 Achados e melhorias (priorização)**

| 🧩 Achado | 📍 Onde ocorreu | 🎯 Impacto | 📈 Frequência | 🔥 Prioridade | ✅ Ação recomendada |
| :---- | :---- | :---- | :---- | :---- | :---- |
| Falta de feedback sonoro ou visual explícito ao disparar convites por e-mail. | T4 | Médio | P1, P3 | 🔥 **Alta** | Inserir um *SnackBar* (balão de aviso na parte inferior) com a mensagem "Convite enviado com sucesso\!". |
| Seleção de categoria obrigatória inicia sem foco padrão no formulário. | T2 | Baixo | P3 | ⚠️ **Média** | Configurar o Dropdown para iniciar com o valor padrão "Outro" previamente selecionado. |
| Erro de digitação de senha não exibe a opção de "olhinho" para revelar o texto. | T1 | Baixo | P3 | 🟦 **Baixa** | Adicionar um botão de visibilidade (suffixIcon) no campo de texto de senha. |

## **✅ Conclusão e decisão**

| Item | Resultado 📝 |
| :---- | :---- |
| **✅ Principais pontos positivos** | Interface intuitiva, criação de eventos simples, checklist funcional e gerenciamento de participantes eficiente.  |
| **⚠️ Principais dificuldades** | Convites poderiam ser mais visíveis e área financeira pode ser expandida.  |
| **🛠️ Top 3 melhorias** | 1\) Melhorar área de convites.  2\) Implementar histórico financeiro.  3\) Adicionar QR Code Pix.  |
| **🟢 Go/No-Go para entrega** | **GO – o sistema atende aos requisitos propostos e apresenta boa usabilidade.**    |
